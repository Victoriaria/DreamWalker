# 巡梦舱 DreamWalker · 产品需求文档（PRD）

> **版本：** V0.1 · 赛题现场版  
> **线上环境：** https://dreamwalker.vercel.app/  
> **代码仓库：** https://github.com/Victoriaria/DreamWalker

---

## 1. 产品定位

巡梦舱 DreamWalker 是一款梦境记录与沉浸式回看 Web 应用。用户写下梦境碎片后，系统生成梦之书、分镜视觉与声景，并通过日历、书架和最后一页留言安放情绪。

**Slogan：** 每个梦，都值得被再次走过。

---

## 2. 目标用户与场景

| 用户 | 场景 |
|------|------|
| 醒来后记得零碎画面的用户 | 快速留下梦碎片，避免遗忘 |
| 需要情绪承接的用户 | 回看梦时看到意象、身体感受与氛围被保留 |
| 长期记录梦境的用户 | 通过书架与日历管理历史梦之书 |

---

## 3. MVP 范围（§3.1）

### 3.1.1 必做能力

- [x] 创建并保存梦境（标题、碎片、情绪、身体感受、氛围色）
- [x] 至少一种非文本录入（语音输入 + 快捷语音兜底）
- [x] 查看历史梦境（梦之书架）
- [x] 日历视图（按日期回看）
- [x] 数据持久化（localStorage）
- [x] 用户/会话隔离（不同旅人名独立存储）
- [x] 情绪相关能力（情绪标记、声景、最后一页留言）

### 3.1.2 核心链路

```
输入旅人名 → 捕捉碎片 → 标记情绪/身体感受/氛围色
→ 生成梦之书 → 巡梦回放 → 最后一页留言 → 书架/日历回看
```

### 3.1.3 非目标（V0.1 不做）

- 云端账号与跨设备同步
- 后端数据库与登录系统
- 第三方 LLM API 依赖（当前为前端规则生成）

## 4. 页面与信息架构

| 路由/视图 | 说明 |
|-----------|------|
| 首页 | 旅人名设置、产品入口 |

## 6. 数据模型（localStorage）

| 键 | 说明 |
|----|------|
| `dreamwalker:currentTraveler` | 当前旅人名 |
| `dreamwalker:data:<traveler>` | 该旅人的梦境数组 |

## 7. 可靠性与回退

详见 [technical-implementation-and-reliability.md](./technical-implementation-and-reliability.md)。

- 空输入拦截
- 超长输入截断
- 语音不可用 → 文字 + 快捷语音
- 图片缺失 → Canvas 动态氛围
- 存储失败 → 提示清理旧梦
