# 技术实现及可靠性机制

## 1. 技术实现概览

巡梦舱 DreamWalker 是一个围绕梦境记录、叙事生成、沉浸回看和长期管理构建的 Web 应用。

当前版本采用前端单页应用实现，核心由 HTML、CSS 和 JavaScript 组成，不依赖后端服务即可完成主要体验。这样可以保证页面打开速度快、部署成本低，也便于在本地或静态站点环境中稳定运行。

产品的主要模块包括：

- 梦境捕捉模块：负责文本碎片、语音入口、情绪、身体感受和氛围选择。
- 梦境理解模块：从用户输入中提取意象、人物、地点、身体感受与核心情绪。
- 叙事生成模块：将多个梦境碎片整理成逐幕梦境叙事。
- 沉浸播放模块：以分镜、视觉、氛围标签和留言卡的形式重新呈现梦境。
- 梦之书架模块：管理历史梦境，支持按梦境宇宙和留言状态筛选。
- 日历回顾模块：按日期查看历史梦境，并从日期进入详情。
- 数据隔离与持久化模块：按旅人身份保存本地数据，刷新或重新打开后仍可回看。

## 2. 核心链路

用户完成一次梦境记录的链路如下：

1. 用户输入多个梦境碎片。
2. 用户选择情绪、强度、身体感受和氛围。
3. 系统分析输入内容，提取梦境中的关键元素。
4. 系统生成梦境标题、逐幕叙事、情绪反馈和视觉映射。
5. 梦境被保存为一本梦之书。
6. 用户进入沉浸式播放，逐幕重新走过梦境。
7. 用户在结尾留言卡中完成情绪表达。
8. 梦境进入书架和日历，可长期回看。

这条链路保证产品不是固定展示内容，而是由用户输入驱动生成最终体验。

## 3. 梦境理解逻辑

梦境理解模块会对用户输入进行轻量分析，重点保留以下信息：

- 关键意象：如海、月亮、门、桥、图书馆、列车、雾、书等。
- 人物关系：如妈妈、朋友、小时候的自己、陌生人、恋人等。
- 地点线索：如海边、学校、车站、房间、森林、图书馆等。
- 身体感受：如胸口发紧、身体很轻、脚步沉、喉咙堵住等。
- 情绪倾向：如思念、治愈、不安、孤独、释然等。

这些信息会参与标题生成、分镜叙事、氛围色、视觉宇宙选择和结尾反馈生成。

## 4. 梦境宇宙映射

为了让视觉体验保持统一，巡梦舱使用“梦境宇宙”概念承接不同情绪类型。

当前核心宇宙包括：

- 记忆鲸落：承接思念、失去、告别、遗憾等情绪。
- 月亮图书馆：承接成长、治愈、怀念、自我对话等情绪。

系统会根据用户输入和情绪选择进行映射。比如出现海、告别、追不上、妈妈、回忆等内容时，更容易进入「记忆鲸落」；出现月亮、图书馆、书、未来、小时候的自己等内容时，更容易进入「月亮图书馆」。

同一宇宙下也会根据不同梦境分配不同封面，避免历史记录出现重复、单调的视觉结果。

## 5. 数据持久化与隔离

巡梦舱使用浏览器本地存储保存数据。

每个旅人都有独立的数据空间，存储键以旅人身份区分，避免不同用户或不同会话之间的数据混杂。

保存内容包括：

- 梦境标题
- 创建时间与更新时间
- 梦境碎片
- 情绪与强度
- 身体感受
- 分镜叙事
- 视觉映射
- 留言内容
- 梦境宇宙

刷新页面或重新打开页面后，历史梦境仍然存在，用户可以从书架或日历继续回看。

## 6. 多模态输入

巡梦舱支持文本输入，也提供语音入口。

在支持浏览器语音识别的环境中，用户可以通过语音把梦境碎片输入到当前片段中。若当前环境暂不支持语音识别，系统会提供快捷语音识别结果入口，让用户仍然能体验除纯文本外的输入方式，并进入完整生成链路。

这样即使在浏览器能力不同的环境中，核心流程仍然可继续完成。

## 7. 视觉呈现机制

视觉层由两部分组成：

- 产品固定视觉资产：用于「记忆鲸落」和「月亮图书馆」等核心梦境宇宙，保证整体视觉质量稳定。
- 本地动态氛围：当图片不可用或外部资源失败时，使用 Canvas 和氛围色生成本地梦境视觉，保证页面不空白。

书架和最近梦境区域会优先展示梦境宇宙图片；若同一屏存在多个相同宇宙的梦，会自动分配不同封面，提高浏览体验。

## 8. 可靠性与恢复策略

### 8.1 空输入处理

如果用户没有输入任何梦境碎片就尝试生成，系统不会进入空结果页面，而是提示用户至少留下一个梦境碎片。

### 8.2 超长输入处理

单条梦境碎片设置了长度上限。若用户输入过长，系统会保留前段内容并提示用户当前内容已达到上限，避免页面卡顿或生成结果失控。

### 8.3 保存失败处理

如果浏览器本地存储空间不足，系统会提示用户删除旧梦后重试，并保留当前输入内容，避免用户刚写下的梦境丢失。

### 8.4 生成异常处理

如果生成过程中出现异常，系统会显示重试入口，并保持用户输入仍在当前页面中，用户可以继续修改或再次生成。

### 8.5 图片加载失败处理

如果图片加载失败，页面不会出现空白卡片。系统会切换到本地动态视觉，并保留梦境标题、情绪、分镜和留言入口。

### 8.6 语音能力不可用处理

如果浏览器不支持语音识别，系统会显示明确提示，并提供快捷语音入口。用户仍然可以通过文字或快捷语音继续完成梦境记录。

### 8.7 路由异常处理

如果用户进入不存在的梦境详情或无效页面，系统会回到可用页面，并提供继续记录或查看书架的入口。

### 8.8 移动端与桌面端适配

页面布局使用响应式结构。桌面端优先展示沉浸式宽屏布局，移动端则收拢为单列，保证主要按钮、输入框、卡片和日历都能继续使用。

## 9. 工程结构

项目主要文件结构：

```text
DreamWalker/
├── index.html
├── styles.css
├── app.js
├── product-assets/
├── docs/
└── README.md
```

其中：

- `index.html`：页面入口。
- `styles.css`：视觉样式、响应式布局和动效。
- `app.js`：状态管理、梦境生成、数据持久化、路由与交互逻辑。
- `product-assets/`：梦境宇宙视觉资产。
- `docs/`：产品说明、展示脚本和技术文档。

## 10. 后续可扩展方向

- 接入真实大模型接口，提高梦境叙事的语言丰富度。
- 接入真实图片生成服务，让每个梦境都生成专属视觉。
- 增加音频氛围层，如海浪、翻书声、低频环境音。
- 增加云端账户体系，实现跨设备同步。
- 增加梦境标签体系和长期情绪趋势分析。
- 支持导出梦之书，形成个人梦境档案。
