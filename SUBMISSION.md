# 巡梦舱 DreamWalker 提交材料

## 100 字以内产品介绍

巡梦舱 DreamWalker 是一款梦境记录与沉浸式回看 Web 应用。用户写下梦境碎片后，系统生成梦之书、分镜视觉与声景，并通过日历、书架和最后一页留言安放情绪。

## 项目一句话

把醒来后零散、模糊、带着情绪的梦，变成一本可以再次走进去的梦之书。

## 项目说明

巡梦舱围绕「记录、管理梦境，并沉浸式承接情绪」构建。用户输入梦境碎片后，系统会识别其中的意象、人物、地点、身体感受和情绪，并生成梦境标题、分镜叙事、视觉氛围、声景回放和最后一页情绪留言。

它解决的不是“写一篇梦境日记”，而是让用户在回看梦境时，能感到自己的情绪被看见、被保存、被安放。

## 必做要求对应

| 要求 | 完成情况 |
| --- | --- |
| 创建并保存梦境 | 已支持标题、多个碎片、情绪、身体感受、氛围色，并保存到本地 |
| 至少一种非文本录入 | 支持语音输入；语音受限时提供快捷语音片段 |
| 查看历史梦境 | 梦之书架可查看、筛选、打开、重新回放 |
| 日历视图 | 可按日期查看当天梦境，并从日历进入详情或回放 |
| 数据持久化 | localStorage 保存，刷新后仍存在 |
| 用户/会话隔离 | 不同旅人名使用不同存储键 |
| 情绪相关能力 | 情绪标记、身体感受、氛围色、声景、最后一页留言 |
| 稳定演示 | 纯前端，无需后端，推荐本地静态服务运行 |

## 运行说明

```bash
cd DreamWalker
python3 -m http.server 5173
```

浏览器打开：

```text
http://localhost:5173
```

无需安装依赖。也可直接打开 `index.html`，但录屏和现场评审更推荐本地静态服务。

## 演示脚本

1. 首页输入旅人名，说明不同旅人有独立梦境空间。
2. 进入「捕捉」，输入多个梦境碎片。
3. 点击「快捷语音」或语音按钮，展示非文本录入。
4. 选择情绪、强度、身体感受和氛围色。
5. 点击「生成梦之书」。
6. 展示生成后的标题、分镜、视觉、输入线索和情绪标签。
7. 点击自动播放，开启声景。
8. 到最后一幕，填写并保存梦后留言。
9. 进入梦之书架，展示历史梦境。
10. 进入日历，点击有梦的日期，从日历打开详情。
11. 刷新页面，展示梦境仍然存在。

## 成果展示文案

「巡梦舱」把梦境碎片转化为一本可回放的梦之书。它不仅记录梦的内容，也记录梦醒后残留的情绪：用户能看见梦里的意象、人物、身体感受和氛围色，并在最后一页留下没有说完的话。

## 录屏重点

- 输入内容要包含人物、地点、意象、身体感受，例如：妈妈、图书馆、列车、海水、胸口发紧。
- 一定展示「快捷语音」或语音输入，避免漏掉非文本录入要求。
- 一定展示日历，并从日历进入详情。
- 一定刷新一次页面，证明数据持久化。
- 最后一幕要保存留言，突出情绪承接。

## 推荐测试输入

```text
我在一座会呼吸的图书馆里醒来，书架像潮水一样退开，妈妈站在月光尽头没有说话。
后来我坐上一列没有车窗的列车，胸口发紧，车票上写着未来两个字，脚下却全是海水。
我想追上她，可走廊越来越长，灯一盏一盏熄灭，只剩海浪声在背后。
```

推荐选择：

```text
情绪：思念
强度：7/10
身体感受：胸口发紧、脚步沉
氛围色：青蓝
```
